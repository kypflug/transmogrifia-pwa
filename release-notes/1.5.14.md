# Library of Transmogrifia 1.5.14

## Fixed

- Pending cloud generation jobs now persist to IndexedDB across page reloads. Sharing a new URL via the OS share sheet no longer clears other in-progress remixes from the queue.
- Proactively clean stale MSAL `interaction.status` keys before `handleRedirectPromise()`, preventing the error path that could wipe accounts and force re-login on desktop PWA re-opens.
- Skip the `ssoSilent()` hidden-iframe attempt (3–6 s timeout) when MSAL has no accounts at all, falling through to a fast `loginRedirect` with `loginHint` instead.
- Batch IndexedDB preference reads with `Promise.all` and pre-warm them in parallel with MSAL initialization, removing ~100 ms from the boot critical path.
- Fixed lightbox not activating on shared article pages by registering the iframe `onload` handler before setting `srcdoc` and deferring setup by one animation frame. Also fixed a null-check error in the shared-viewer iframe reference.

## Changed

- Updated `@kypflug/transmogrifier-core` to 0.1.34.
- All `/api` routes are proxied through the Vite dev server so cloud API calls (queue, share, share-link resolution) work from `localhost` without CORS errors.

## Validation

- Full `npm run build` passes (TypeScript + Vite production build).
